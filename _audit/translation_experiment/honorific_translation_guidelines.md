# 《死神与少女 Bookish》敬语与称呼翻译准则

> 日期：2026-06-05  
> 范围：基于 `bookish/reading_order/*.md` 与关键章节片段遍历。  
> 用途：供后续 LLM 翻译实验、人工审校和 accepted block 回填使用。  
> 目标读者假设：简体中文 ACGN/VN 读者，可以接受一定日式称谓，但不希望整篇变成机械音译。  

---

## 一、总原则

本作的敬语系统不能用“全部归化”或“全部音译”解决。推荐采用 **中度保留的 ACGN 策略**：

1. **保留有剧情功能的称呼差异**  
   `日生先輩`、`お嬢`、`お嬢さん`、`蒼さん→蒼`、`お嬢→紗夜` 这类称呼直接构成人际关系和剧情转折，必须在中文中可见。

2. **不把 `さん` 机械译成“先生”**  
   `さん` 的语义跨度很宽。`千代さん`、`光さん`、`蒼さん`、`臥待さん` 不能使用同一译法。

3. **不把所有亲昵称呼都机械音译**  
   `ちゃん` 可以在 ACGN 语境下译作“酱”，但只用于确实需要日式亲昵称呼感的关系，不能因为看到 `ちゃん` 就自动套。

4. **敬体优先通过语气和句式体现**  
   中文没有 `です/ます` 的形态系统。纱夜和千代的敬体，要靠“请、您、较完整句式、克制语气、少用随意语气词”来传达。

5. **称呼转换优先于词面漂亮**  
   如果某个称呼变化是剧情里程碑，中文必须保留“变化”本身，即便局部词面不如纯归化自然。

## 四、`さん` 的分类型处理

### 4.1 纱夜 → 苍：`蒼さん → 蒼`

原文：

```text
遠野　紗夜: 「あの、それで蒼さん」
蒼: 「……『さん』？」
蒼: 「私の名を『蒼』だと名付けたのはお前ではなかったのか」
```

这里 `さん` 是剧情元语言。苍不是讨厌尊敬，而是拒绝纱夜在自己亲手命名后仍加一个社交距离后缀。

推荐译法：

```text
远野纱夜：「那个，所以苍先生……」
苍：「……『先生』？」
苍：「给我取名叫『苍』的人，不就是你吗？」
```

后续纱夜对苍应改为“苍”，不再“苍先生”。  
这个场景允许用“先生”，不是因为 `さん = 先生`，而是因为中文需要一个可被摘掉的距离标记。

### 4.2 纱夜 → 千代：`千代さん`

不推荐默认把后缀显性化为“千代先生/千代小姐/千代女士/千代同学/千代桑”。理由：

- 千代不是教师/专业人士。
- “先生/小姐/女士”会让中文读者误判年龄、性别和社会身份。
- “同学”会把千代拉回普通校园同辈关系，削弱其非现实存在感。
- “桑”能保留日式后缀，但会让整篇语气偏二创/同人腔，不符合本项目的中度保留策略。
- 当前译文中“千代先生/千代小姐/千代女士/千代同学”混用已经说明机械处理会失控。

推荐策略：

- `千代さん` 默认译作“千代”。这里的“千代”不是亲昵称呼，而是中文里对 `さん` 的关系化省略。
- 对白中多用“千代” + 礼貌句式：`千代，请不要消失……`
- 不把 `さん` 固定落成一个中文称谓；纱夜对千代的敬意主要通过“请、较完整句式、克制语气”表达。
- `您` 可用于初遇、郑重请求、道歉、说明等需要保持距离的句子；亲密推进、情绪升高、或中文读起来过度僵硬时，可自然转为“你”或省略第二人称。
- 必要的正式介绍可按上下文临时处理，但“千代先生/小姐/女士/同学/桑”不作常规称呼。

### 4.3 千代 → 纱夜：`お嬢さん → 紗夜さん`

这是千代线的关键转换。

默认：

- `お嬢さん` → “小姐”
- `紗夜さん` → “纱夜”

对应处理：

```text
千代：「纱夜」
第一次被他直接叫了名字，我抬起了头。
```

这样比“纱夜小姐”更能保留转换功能。原文仍有 `さん`，但中文重点是从身份称呼“小姐”切到私人名字“纱夜”。千代自身的礼貌保留在后续句式里，例如“请不要太悲伤”“我会一直守望着您”。

### 4.4 纱夜 → 光：`日生先輩 → 光さん`

关系推进前：

- `日生先輩` → “日生前辈”

关系推进后：

- 私下、恋人语境：`光さん` → “光”
- 正式对长辈介绍：可译“日生光先生”或“光先生”，但只限正式场合。

关键是保留名字转换：

```text
远野纱夜：「终于叫我的名字了呢。明明一直叫我『小姐』。」
```

不要在同一条线中混用“前辈/学长”，也不要把恋人私下的 `光さん` 全部译成“光先生”，那会显得像商务称呼。

### 4.5 成人/社会关系中的 `さん`

可按中文身份译：

- `臥待さん` → “卧待先生”
- `司書さん` → “图书管理员小姐/图书管理员”
- `看護士さん` → “护士小姐/护士”
- `日生紫さん` 等成人女性 → “紫女士/紫夫人”，按上下文

这类 `さん` 不承担 ACGN 后缀游戏，优先自然中文。

## 五、`ちゃん` 的处理

### 5.1 夏帆 → 纱夜：`紗夜ちゃん`

推荐译法：**纱夜酱**。

理由：

- 夏帆全书稳定使用 `紗夜ちゃん`。
- 它是夏帆活泼、亲密、略撒娇的声音核心。
- 本项目面向日文 ACGN 文本，读者可接受“酱”。

但要求：

- 只用于夏帆/熟人亲昵称呼，不全局滥用。
- 不把叙述层的普通名字也改成“酱”。
- 如果某个场景需要更正式或更痛苦的语气，可个案审查是否改为“纱夜”。

### 5.2 光/其他人使用 `ちゃん`

光说 `紗夜ちゃん` 时多半带玩笑、调侃、过度亲昵。可译“纱夜酱”，以显示他故意越界的轻佻。

长辈对纱夜的 `紗夜ちゃん` 可以用“纱夜酱”或“小纱夜”，视该角色是否日式/生活化。若要保持 ACGN 一致，优先“纱夜酱”。

---

## 六、`先輩` 的处理

本项目推荐统一为 **前辈**。

理由：

- 原作是日文 ACGN/VN，中文读者对“前辈”接受度高。
- “学长/学姐”会把性别和中国校园语境带得过重。
- 纱夜对 `日生先輩`、`桐島先輩` 的称呼是她敬体人格的一部分。

规则：

- `日生先輩` → 日生前辈
- `桐島先輩` → 桐岛前辈
- 单独 `先輩` → 前辈
- 不与“学长”混用

例外：若某个非主线学生群众对白明显是普通校园闲聊，仍建议优先“前辈”，以保持全书统一。

---

## 七、`君` 的处理

分两类：

1. **名称后缀**：保留“君”
   - `七葵君` → 七葵君
   - `桐島君` → 桐岛君
   - `夏目君` → 夏目君

2. **第二人称代词**：译“你”
   - 光对纱夜的 `君` → 你
   - 臥待等成人对年轻人的 `君` → 你 / 你这孩子，按语气

`君` 是 ACGN 中可保留的轻日式称呼，语感比“同学”更不受现实校园限制。

---

## 八、`様` 的处理

`様` 不能统一译“大人”。

| 原文 | 推荐 |
|---|---|
| `お嬢様` | 大小姐 |
| `紗夜お嬢様` | 纱夜大小姐 |
| `奥様` | 夫人 |
| `紫様` | 紫夫人 / 紫女士 |
| `王子様` | 王子殿下 / 王子 |
| `お姫様` | 公主 / 公主殿下，视童话层级 |
| `神様` | 神明 |

幻想/童话层可以比现实层更保留“殿下/大人”等称谓，但现实校园和家族关系要自然。

## 九、纱夜的敬体表达

纱夜的敬语不是“乖巧”，而是防御结构。中文处理原则：

1. 对前辈、陌生人、苍初遇、父亲、成人：优先“您”。
2. 对夏帆：可用“你”，因为夏帆是少数亲密平辈。
3. 对哥哥：不用“您”，但保持句式端正；不要过度口语化。
4. 强硬对白也保持礼貌外壳。

例：

```text
私は貴方の人形ではありません！！
```

应保留“礼貌形式下的反抗”，可译：

```text
我不是您的玩偶！！
```

而不是：

```text
我才不是你的玩具！！
```

后者更口语、更爆发，但少了纱夜的外壳。

## 十一、当前译文中应避免的混用

从现有 `bookish_zhcn` 可见几个风险：

1. `日生先輩` 同时出现“日生前辈/日生学长”。  
   准则：统一“日生前辈”。

2. `桐島先輩` 同时可能被处理成“桐岛前辈/桐岛学长”。  
   准则：统一“桐岛前辈”。

3. `千代さん` 出现“千代先生/千代小姐”。  
   准则：不要机械性别化；默认“千代”，靠敬语句式补距离。

4. `お嬢` 出现“小姐/大小姐”。  
   准则：`お嬢` = 小姐；`お嬢様` = 大小姐。

5. 恋人后的 `光さん` 被译成“光先生”。  
   准则：私下应译“光”；正式介绍时才可“光先生”。

## 十二、给 LLM 实验的硬约束

后续 LLM 生成候选译文时，应在 prompt 中加入：

1. 不得把所有 `さん` 统一译作“先生”。
2. 不得把 `先輩` 译法在“前辈/学长”之间摇摆。
3. `お嬢 → 紗夜`、`蒼さん → 蒼`、`お嬢さん → 紗夜さん` 必须保持可见转换。
4. 纱夜的敬体要通过“您/请/完整句式/克制语气”体现。
5. `ちゃん` 可以译“酱”，但只用于角色亲昵称呼，不全局滥用。
6. 如果为自然中文省略了后缀，必须在 review block 的 `note` 中说明省略理由。

建议在 parallel review 的 `tags` 中加入：

- `honorific`
- `address-shift`
- `politeness`
- `acgn-register`
- `name-vs-title`

---

## 十三、结论

本项目最适合的敬语策略是：

> **保留 ACGN 读者能自然识别的关系标记，归化普通社交后缀，显化所有剧情性称呼转换。**

也就是说：

- “前辈”“酱”“君”可以用，但要有角色功能。
- “先生”不能泛滥，只在真的需要中文可摘除称呼或成人正式语境时使用。
- “您”不是敬体万能解，但纱夜对陌生人、前辈、苍初遇的距离感需要它。
- 重要的不是每个日文后缀都有固定中文词，而是读者能感到关系如何移动。

这套准则应优先于单个 LLM 对“自然中文”的即兴判断。
