# 现有中文译文的纯净评估

> 日期：2026-06-05  
> 范围：`bookish/reading_order/*.md` 与 `bookish_zhcn/reading_order/*.md` 对照取样。  
> 约束：本评估没有阅读现有审校报告、模型报告或 `_audit` 下的既有结论。  
> 目的：从日文原文和现有中文译文出发，判断后续最值得改进的方向。

---

## 一、总体判断

现有 `bookish_zhcn` 已经不是不可读的机器稿。它的情节覆盖、基本语义和大多数场景推进是可用的，很多地方也已经比早期 legacy 文本自然。

但如果要把它作为后续 LLM 翻译实验的基准，它还需要先做一轮 **洁净化与关系系统校准**。目前最大的问题不是“中文不够华丽”，而是：

1. 称呼和敬语系统不稳定，且有些地方明显误读日文称谓。
2. 引号、书名号、内嵌引用和多行台词存在较多成稿层面的脏点。
3. 纱夜的敬体人格在部分章节被削弱，变得过于普通口语。
4. 高频术语没有统一，读者空间记忆会被打断。
5. 少数句子残留了译者不确定痕迹或明显误译。

这些问题会影响后续实验：LLM 很容易模仿当前译文的混用，而不是回到日文原文重新判断。

---

## 二、第一优先级：成稿洁净度

### 2.1 引号和内嵌引用错配

序章童话段和后期真相揭示段里，原文经常使用 `「」` 包住对白，再用 `『』` 表示书中话语或概念。当前中文在多行切分后出现了明显错配。

例：

```text
远野十夜: 「我想出笼去』——这是笼中鸟的口头禅。」
```

这里开头是中文外层引号 `「`，结尾却是内层 `』`。读者会以为这是缺字或排版错误。

另一个例子：

```text
远野十夜: 「如果您想离开笼子，我就帮您实现这个愿望，」
远野十夜: 「帮您实现。」
```

日文原文只是因为脚本换行把 `差しあげましょう` 切开，中文不应重复成两句。

建议：

- 建立一个专门的 quote cleanup pass。
- 不只看单行括号平衡，还要按“同一说话人连续台词”重组判断。
- 童话、书中书、真相说明、概念强调四类场景要单独抽查。

### 2.2 译者不确定痕迹

当前正文中存在类似：

```text
面对我的问题，他露出一副意外（或不以为然）的表情。
```

这种括号不是文学表达，而是译者在犹豫。应改成确定译法。

对应日文是：

```text
私の質問に彼は心外だという顔をする。
```

可改为：

```text
面对我的问题，他露出一副不服气的表情。
```

或：

```text
听到我的问题，他像是受了冤枉似的看着我。
```

### 2.3 明显误读

例：

```text
日生光: 「什么够不够？」
```

对应日文：

```text
日生 光: 「何が？」
```

这里应只是：

```text
日生光: 「什么？」
```

或：

```text
日生光: 「你指什么？」
```

这种错误可以通过“怪异中文短语”扫描先抓一批。

## 三、第二优先级：称呼系统

### 3.1 `先輩` 的混用

当前出现了：

- `日生前辈`
- `日生学长`
- `光前辈`
- `光先生`

其中 `光先輩` 被译成“光先生”是明显偏离。`先輩` 的关系是校园/前后辈，不是成人职业或礼貌称号。

例：

```text
今日は確か光先輩と一緒にいるのではなかっただろうか。
```

现译：

```text
今天他应该和光先生在一起才对。
```

建议至少改为：

```text
今天他应该和光前辈在一起才对。
```

如果上下文一直以姓称呼，则可统一为“日生前辈”。关键是不要把 `先輩` 译成“先生”。

### 3.2 `光さん` 的恋人称呼

日生线后期，原文从 `日生先輩` 切到 `光さん`，这是关系推进的核心信号。现译大量使用“光先生”，读起来像商务称呼或对成人男性的敬称，削弱了恋人称呼的亲密感。

例：

```text
遠野　紗夜: 「…………光さん」
```

现译：

```text
远野纱夜: 「…………光先生」
```

建议私下恋人语境改为：

```text
远野纱夜: 「…………光」
```

正式见祖母、介绍身份时可以斟酌“日生光先生”，但不能把后期所有 `光さん` 都机械译为“光先生”。

### 3.3 `お嬢 / お嬢さん / お嬢様`

当前 `お嬢さん` 被多次译成“大小姐”，例如千代对纱夜喊：

```text
千代: 「七葵くーん！！　……あ！　お嬢さん！　お嬢さん！！」
```

现译：

```text
千代: 「七葵君——！！……啊！大小姐！大小姐！！」
```

`お嬢さん` 在千代口中更像“小姐”，不是家政/贵族式的“大小姐”。“大小姐”应留给 `お嬢様` 或日生故意夸张调戏的场合。

建议：

- `お嬢`：小姐
- `お嬢さん`：小姐
- `お嬢様`：大小姐

### 3.4 `千代さん`

当前同时存在“千代先生”和“千代小姐”，且在同一角色线里频繁摇摆。这会让读者误判千代的性别、年龄和社会身份。

更稳妥的策略是：

- 纱夜对白中多用“千代”。
- 通过“您 / 请 / 完整句式”保留礼貌。
- 叙述中也尽量避免“千代先生/千代小姐”来回切。

例如：

```text
远野纱夜: 「千代先生，您要不要来我这边？」
```

可试为：

```text
远野纱夜: 「千代，您要不要到我这边来？」
```

## 四、第三优先级：术语统一

`時計塔` 现在至少出现了四种译法：

- 时计塔
- 钟塔
- 钟楼
- 时钟塔

甚至在序章同一小段中出现：

```text
女学生A: 「呐呐，听说了吗？　钟楼的传闻」
听到『钟塔』这个词，我微微一动。
时计塔和死神——到底是什么样的传闻呢？
```

这会直接打断读者对核心地点的记忆。

建议先做术语决策：

- 若追求自然中文：统一为“钟楼”。
- 若强调塔状建筑与象征性：统一为“钟塔”。
- 若刻意保留日式 ACGN 味：可统一“时计塔”，但要接受它在现代中文里较生硬。

我个人从中文阅读体验出发，倾向 **钟塔**：它保留 `塔` 的形象，又比“时计塔”自然。

---

## 五、第四优先级：纱夜的语体

纱夜的敬体不是装饰，而是她人格的一部分。当前部分章节把她翻得过于普通口语。

例：

```text
遠野　紗夜: 「何を読んでいるのですか？」
```

现译：

```text
远野纱夜: 「你在读什么书？」
```

这里对哥哥不必用“您”，但“你在读什么书”太日常、太平。可改为：

```text
远野纱夜: 「哥哥在读什么书呢？」
```

或：

```text
远野纱夜: 「您在读什么书呢？」
```

若项目不希望兄妹间用“您”，第一种更好。

另一个例子：

```text
遠野　紗夜: 「……読んで下さいませんか？」
```

现译：

```text
远野纱夜: 「……能念给我听吗？」
```

可改为：

```text
远野纱夜: 「……可以读给我听吗？」
```

或更纱夜一点：

```text
远野纱夜: 「……可以请哥哥读给我听吗？」
```

再如：

```text
遠野　紗夜: 「……ええ。また休んでしまいました。貴方は？」
```

现译：

```text
远野纱夜: 「……嗯。我又翘课了。你呢？」
```

“翘课”暗示主动逃学，与原文病弱、无奈的 `休んでしまいました` 不合。可改为：

```text
远野纱夜: 「……是的。我又没能去学校。您呢？」
```

如果此时她与苍关系已经较近，也可以不用“您”，但“翘课”应避免。

## 六、第五优先级：叙事层的文学性与准确度

当前叙事多数能读，但有些句子过于直译，或提前把情感说重。

例：

```text
私はこの気持ちが何なのか分からない。
自分とはまるで違う彼女のことを思っていた。
```

现译：

```text
我不知道这种心情是什么。
我思念着和我完全不同的她。
```

`思念` 在中文里带有较明确的怀恋/恋慕色彩，可能过早替原文下结论。可改为：

```text
我不知道这种心情是什么。
我想着那个与我截然不同的她。
```

又如：

```text
大好きな兄、大好きな物語、私の時間が大好きなもので流れていく。
```

现译：

```text
我最爱的哥哥，最爱的故事，我的时间流淌着最爱之物。
```

“时间流淌着最爱之物”较生硬。可改为：

```text
我最爱的哥哥，最爱的故事。我的时间，被这些最喜欢的东西填满。
```

这类不必全书重写，但适合在关键章节和情绪高潮段做精修。

## 七、可机械先行的扫描项

建议先建立一批自动扫描或半自动清单：

1. `光先生`、`光前辈`、`日生学长`、`日生前辈` 的上下文对齐。
2. `千代先生`、`千代小姐`、`纱夜小姐` 的上下文对齐。
3. `大小姐` 是否来自 `お嬢様`，还是误译了 `お嬢 / お嬢さん`。
4. `时计塔 / 钟塔 / 钟楼 / 时钟塔` 统一。
5. `谢谢你们` 是否真的有复数对象。
6. `（或`、`暂译`、`原文`、`不以为然` 等译者痕迹。
7. 中文奇怪短语：如“什么够不够”。
8. 书中书和概念引用的 `「」`、`『』`、`‘’` 错配。

这些清理完成后，再让 LLM 做风格优化会更可靠。

---

## 八、建议的实验顺序

1. **硬错误清理**
   - 引号、残留注释、明显误译、高频复数误用。

2. **称呼与术语统一**
   - 先确定固定表，再回填正文。

3. **纱夜语体校准**
   - 选序章、日生线、苍线、千代线各一段，建立 accepted samples。

4. **角色线精修**
   - 日生线优先处理 `日生先輩 → 光さん`。
   - 千代线优先处理 `お嬢さん → 紗夜さん`。
   - 苍线优先处理病弱、静默、终结相关叙事。

5. **文学段落优化**
   - 只在高密度象征段落做精修，不要全书泛泛润色。

---

## 九、结论

现有中文译文最值得改进的不是“把句子变得更美”，而是先恢复原文的 **关系精度、称呼秩序和成稿洁净度**。

如果直接让 LLM 在当前文本上润色，它很可能会继续沿用“光先生 / 日生学长 / 大小姐 / 千代小姐 / 时计塔和钟塔混用”这些问题。更好的做法是先完成一轮清洗和称呼统一，再进入并排实验。

我会把后续实验的重点放在：

- 称呼转换是否可见。
- 纱夜敬体是否稳定。
- 中文是否自然，但不抹平 ACGN/VN 关系标记。
- 关键象征段落是否保留原文的短句节奏和童话感。
